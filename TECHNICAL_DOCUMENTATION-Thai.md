# เอกสารทางเทคนิค - เว็บไซต์พอร์ตโฟลิโอ NextJS

## 1. ภาพรวม

นี่คือเว็บไซต์พอร์ตโฟลิโอที่ทันสมัยและมีปฏิสัมพันธ์ ถูกสร้างด้วย **Next.js 15** และ **React 19** เพื่อนำเสนอผลงานและทักษะ แอปพลิเคชันนี้มีดีไซน์แนวธีมอวกาศพร้อมแอนิเมชันขั้นสูง เลย์เอาต์ที่รองรับทุกขนาดหน้าจอ และองค์ประกอบที่มีปฏิสัมพันธ์

### ฟีเจอร์หลัก:
- **Single Page Application (SPA)** พร้อมการเลื่อนหน้าจออย่างนุ่มนวล
- **สลับธีมมืด/สว่าง** ด้วย CSS custom properties
- **แอนิเมชันขั้นสูง** โดยใช้ Framer Motion
- **ดีไซน์ตอบสนอง** เหมาะสมกับทุกอุปกรณ์
- **เอฟเฟกต์เคอร์เซอร์แบบโต้ตอบ** และองค์ประกอบภาพแนวจักรวาล
- **UI แบบ Glass morphism** พร้อม backdrop filters
- **ประสิทธิภาพที่ปรับแต่ง** ด้วย React.memo และ useMemo

### สถาปัตยกรรม:
- **เฟรมเวิร์ก**: Next.js 15 พร้อม App Router
- **การเรนเดอร์**: Client-side rendering (CSR) ด้วย 'use client' directive
- **การจัดรูปแบบ**: TailwindCSS + DaisyUI + Custom CSS
- **แอนิเมชัน**: Framer Motion
- **ไอคอน**: FontAwesome + React Icons
- **TypeScript**: Type safety เต็มรูปแบบ

## 2. ไลบรารี/เฟรมเวิร์กที่ใช้

### เฟรมเวิร์กและไลบรารีหลัก:
- **Next.js 15.3.2** - เฟรมเวิร์ก React พร้อม App Router
- **React 19.0.0** - ไลบรารี UI พร้อมฟีเจอร์ล่าสุด
- **TypeScript 5** - Type safety และประสบการณ์การพัฒนา

### การจัดรูปแบบและ UI:
- **TailwindCSS 4.1.4** - เฟรมเวิร์ก CSS แบบ Utility-first
- **DaisyUI 5.0.30** - ไลบรารีคอมโพเนนต์สำหรับ TailwindCSS
- **Custom CSS** - แอนิเมชันและเอฟเฟกต์แนวธีมอวกาศ

### แอนิเมชันและการโต้ตอบ:
- **Framer Motion 12.23.6** - แอนิเมชันและท่าทางขั้นสูง
- **React Icons 5.5.0** - ไลบรารีไอคอน
- **FontAwesome 6.7.2** - ไอคอนระดับมืออาชีพ

### เครื่องมือการพัฒนา:

- **PostCSS** - การประมวลผล CSS

## 3. โครงสร้างคอมโพเนนต์

### โครงสร้างแอปพลิเคชันหลัก:
```
src/
├── app/
│   ├── layout.tsx          # เลย์เอาต์หลักพร้อม metadata
│   ├── page.tsx           # คอมโพเนนต์หน้าหลัก
│   ├── globals.css        # สไตล์ global และตัวแปร CSS
│   └── css/               # ไฟล์ CSS เฉพาะคอมโพเนนต์
└── components/
    ├── Navigation.tsx     # เมนูนำทางแบบคงที่พร้อมปุ่มสลับธีม
    ├── Background.tsx     # พื้นหลังแอนิเมชันพร้อมดาวตก
    ├── HeroSection.tsx    # ส่วนหน้าแรกพร้อมแอนิเมชัน
    ├── SkillsSection.tsx # แสดงทักษะพร้อมไอคอน
    ├── ExperienceSection.tsx # ไทม์ไลน์ประสบการณ์ทำงาน
    ├── ContactSection.tsx # ฟอร์มติดต่อและข้อมูล
    ├── ImgWorkSection.tsx # แกลเลอรีสภาพแวดล้อมการทำงาน
    └── ProjectSection/
        ├── ProjectsSection.tsx # แท็บโปรเจกต์และคอนเทนเนอร์
        ├── ProjectCard.tsx    # การ์ดโปรเจกต์ส่วนตัว
        └── WorkProjectCard.tsx # การ์ดโปรเจกต์งาน
```

### หน้าที่ความรับผิดชอบของคอมโพเนนต์:

#### **Navigation.tsx**
- **วัตถุประสงค์**: แถบเมนูนำทางแบบคงที่พร้อมเอฟเฟกต์ glass morphism
- **ฟีเจอร์**: 
  - เมนู drawer สำหรับมือถือแบบตอบสนอง
  - สวิตช์สลับธีม
  - การนำทางแบบเลื่อนนุ่มนวล
  - Glass morphism พร้อม backdrop blur
- **สถานะ**: ไม่มี (stateless)
- **Props**: `toggleTheme`, `scrollToSection`

#### **HeroSection.tsx**
- **วัตถุประสงค์**: ส่วนหน้าแรกพร้อมข้อความแอนิเมชันและเอฟเฟกต์จักรวาล
- **ฟีเจอร์**:
  - แอนิเมชันข้อความทีละตัวอักษร
  - พื้นหลังเครือข่ายประสาท
  - แอนิเมชันอนุภาคควอนตัม
  - วงแหวนโคจร
  - ปุ่มแบบโต้ตอบพร้อมเอฟเฟกต์จักรวาล
- **สถานะ**: ไม่มี (stateless)
- **Props**: `theme`, `scrollToSection`

#### **SkillsSection.tsx**
- **วัตถุประสงค์**: แสดงทักษะพร้อมไอคอนแอนิเมชันและหมวดหมู่
- **ฟีเจอร์**:
  - ไอคอน FontAwesome พร้อมแอนิเมชัน hover
  - แอนิเมชันเข้าแบบทีละชุด
  - เอฟเฟกต์ประกายลอย
  - วงรีแอนิเมชัน SVG
- **สถานะ**: ไม่มี (stateless)
- **Props**: `theme`

#### **ExperienceSection.tsx**
- **วัตถุประสงค์**: ไทม์ไลน์ประสบการณ์ทำงานพร้อมการ์ดแอนิเมชัน
- **ฟีเจอร์**:
  - ไทม์ไลน์พร้อมจุดและเส้นเรืองแสง
  - แอนิเมชันการ์ดลอย
  - แอนิเมชันข้อความแบบทีละชุด
  - การ์ดแบบ glass morphism
- **สถานะ**: ไม่มี (stateless)
- **Props**: ไม่มี

#### **ProjectsSection.tsx**
- **วัตถุประสงค์**: แสดงโปรเจกต์พร้อมอินเทอร์เฟซแท็บ
- **ฟีเจอร์**:
  - สลับแท็บระหว่างโปรเจกต์ส่วนตัว/โปรเจกต์งาน
  - เลย์เอาต์กริดพร้อมดีไซน์ตอบสนอง
  - การจัดการข้อมูลโปรเจกต์
- **สถานะ**: `activeTab` (personal/projects)
- **Props**: ไม่มี

#### **ContactSection.tsx**
- **วัตถุประสงค์**: ฟอร์มติดต่อและแสดงข้อมูล
- **ฟีเจอร์**:
  - ฟอร์มติดต่อพร้อมการตรวจสอบ
  - การผูกรวม mailto
  - ลิงก์โซเชียลมีเดีย
  - เอฟเฟกต์พื้นหลังจักรวาล
- **สถานะ**: `formData` (name, email, message)
- **Props**: ไม่มี

## 4. การไหลของข้อมูล

### การจัดการสถานะ:
แอปพลิเคชันใช้ **local component state** ด้วย React hooks:

1. **สถานะหน้าหลัก** (`page.tsx`):
   - `theme`: 'dark' | 'light' - ควบคุมการสลับธีม
   - `mousePosition`: {x, y} - ติดตามเคอร์เซอร์สำหรับเอฟเฟกต์แบบโต้ตอบ
   - `sectionRef`: อ้างอิงสำหรับการเลื่อนหน้าจอ

2. **สถานะส่วนโปรเจกต์** (`ProjectsSection.tsx`):
   - `activeTab`: 'personal' | 'projects' - ควบคุมการสลับแท็บ

3. **สถานะฟอร์มติดต่อ** (`ContactSection.tsx`):
   - `formData`: {name, email, message} - ค่าอินพุตฟอร์ม

### รูปแบบการไหลของข้อมูล:
```
การโต้ตอบของผู้ใช้ → ตัวจัดการเหตุการณ์ → อัปเดตสถานะ → คอมโพเนนต์เรนเดอร์ใหม่ → อัปเดต UI
```

### ระบบธีม:
- **CSS Custom Properties** สำหรับตัวแปรธีม
- **แอตทริบิวต์ data-theme** บน document root
- **การสลับสีแบบไดนามิก** โดยไม่ต้องรีโหลดหน้า

## 5. การผูกรวม API

### การใช้ API ปัจจุบัน:
- **ไม่มี** - นี่คือเว็บไซต์พอร์ตโฟลิโอแบบ static
- **บริการภาพภายนอก**: ใช้ `picsum.photos` สำหรับภาพตัวอย่าง
- **การผูกรวม Mailto**: ฟอร์มติดต่อเปิดโปรแกรมอีเมลเริ่มต้น

### การผูกรวม API ที่เป็นไปได้:
- **ฟอร์มติดต่อ**: สามารถผูกรวมกับบริการอีเมล (SendGrid, Nodemailer)
- **Analytics**: Google Analytics หรือเทียบเท่า
- **CMS**: Headless CMS สำหรับการจัดการเนื้อหาแบบไดนามิก

## 6. วิธีการทำงาน (ทีละขั้นตอน)

### การเริ่มต้นแอปพลิเคชัน:
1. **Next.js App Router** โหลด `layout.tsx`
2. **Root Layout** ตั้งค่า:
   - โครงสร้าง HTML พร้อมฟอนต์ Geist
   - Metadata สำหรับ SEO
   - ตัวแปร CSS แบบ global
3. **หน้าหลัก** (`page.tsx`) เรนเดอร์ด้วย 'use client' directive

### ลำดับการเรนเดอร์คอมโพเนนต์:
1. **คอมโพเนนต์พื้นหลัง**:
   - สร้างดาวตกด้วยตำแหน่งสุ่ม
   - สร้างพื้นหลังทุ่งดาวแอนิเมชัน
   - เรนเดอร์องค์ประกอบจักรวาล

2. **คอมโพเนนต์นำทาง**:
   - เรนเดอร์เมนูนำทาง glass morphism แบบคงที่
   - ตั้งค่าเมนู drawer สำหรับมือถือ
   - ใช้งานฟังก์ชันสลับธีม

3. **ส่วน Hero**:
   - แอนิเมชันข้อความทีละตัวอักษร
   - เรนเดอร์พื้นหลังเครือข่ายประสาท
   - สร้างเอฟเฟกต์อนุภาคควอนตัม
   - แสดงปุ่มแบบโต้ตอบ

4. **ส่วนทักษะ**:
   - แอนิเมชันไอคอนทักษะแบบเข้าทีละชุด
   - เรนเดอร์เอฟเฟกต์ประกายลอย
   - แสดงทักษะในเลย์เอาต์แบ่งหมวดหมู่

5. **ส่วนประสบการณ์**:
   - สร้างไทม์ไลน์พร้อมองค์ประกอบเรืองแสง
   - แอนิเมชันการ์ดประสบการณ์พร้อมเอฟเฟกต์ลอย
   - เรนเดอร์ประวัติการทำงานพร้อมแอนิเมชัน

6. **ส่วนโปรเจกต์**:
   - จัดการสถานะแท็บสำหรับโปรเจกต์ส่วนตัว/โปรเจกต์งาน
   - เรนเดอร์การ์ดโปรเจกต์ในกริดตอบสนอง
   - จัดการการแสดงข้อมูลโปรเจกต์

7. **ส่วนติดต่อ**:
   - จัดการสถานะฟอร์ม
   - เรนเดอร์ข้อมูลติดต่อ
   - จัดการการส่งฟอร์มผ่าน mailto

### ระบบแอนิเมชัน:
1. **Framer Motion** จัดการแอนิเมชันทั้งหมด
2. **Intersection Observer** กระตุ้นแอนิเมชันเมื่อเลื่อนหน้าจอ
3. **CSS Animations** จัดการเอฟเฟกต์พื้นหลัง
4. **React State** จัดการองค์ประกอบแบบโต้ตอบ

### ระบบธีม:
1. **สถานะธีม** เก็บในคอมโพเนนต์หลัก
2. **CSS Custom Properties** อัปเดตผ่านแอตทริบิวต์ data-theme
3. **คอมโพเนนต์ทั้งหมด** ใช้ตัวแปร CSS สำหรับสี
4. **การเปลี่ยนแบบนุ่มนวล** ระหว่างธีม

## 7. การปรับปรุงที่เป็นไปได้

### การเพิ่มประสิทธิภาพ:
1. **การเพิ่มประสิทธิภาพภาพ**:
   - เปลี่ยนภาพตัวอย่างเป็นภาพหน้าจอโปรเจกต์จริง
   - ใช้การเพิ่มประสิทธิภาพภาพของ Next.js
   - เพิ่ม lazy loading สำหรับภาพ

2. **Code Splitting**:
   - ใช้ dynamic imports สำหรับคอมโพเนนต์ขนาดใหญ่
   - แยกไฟล์ CSS ตามคอมโพเนนต์
   - ใช้ React.lazy สำหรับคอมโพเนนต์ที่ไม่สำคัญ

3. **การเพิ่มประสิทธิภาพ Bundle**:
   - วิเคราะห์ขนาด bundle ด้วย webpack-bundle-analyzer
   - ลบ dependencies ที่ไม่ได้ใช้
   - เพิ่มประสิทธิภาพ imports ของ Framer Motion

### การปรับปรุงฟังก์ชันการทำงาน:
1. **ฟอร์มติดต่อ**:
   - ผูกรวมกับบริการอีเมล (SendGrid, Resend)
   - เพิ่มการตรวจสอบฟอร์มและการจัดการข้อผิดพลาด
   - ใช้สถานะสำเร็จ/ข้อผิดพลาด

2. **การจัดการเนื้อหา**:
   - เพิ่มการผูกรวม headless CMS (Strapi, Contentful)
   - ใช้การจัดการโปรเจกต์แบบไดนามิก
   - เพิ่มส่วนบล็อก

3. **SEO และ Analytics**:
   - ใช้ structured data (JSON-LD)
   - เพิ่ม Google Analytics
   - เพิ่มประสิทธิภาพ meta tags และ Open Graph

### ประสบการณ์ผู้ใช้:
1. **การเข้าถึง**:
   - เพิ่มป้ายกำกับและบทบาท ARIA
   - ใช้การนำทางด้วยคีย์บอร์ด
   - เพิ่มการจัดการโฟกัส
   - ตรวจสอบความตัดกันของสี

2. **ประสิทธิภาพ**:
   - เพิ่มสถานะการโหลด
   - ใช้ skeleton screens
   - เพิ่มประสิทธิภาพแอนิเมชัน
   - เพิ่ม error boundaries

3. **ประสบการณ์มือถือ**:
   - เพิ่มประสิทธิภาพการโต้ตอบแบบสัมผัส
   - ปรับปรุงการนำทางบนมือถือ
   - เพิ่มท่าทางปัด
   - เพิ่มประสิทธิภาพสำหรับมือถือ

### การปรับปรุงทางเทคนิค:
1. **การทดสอบ**:
   - เพิ่ม unit tests ด้วย Jest/React Testing Library
   - ใช้ E2E tests ด้วย Playwright
   - เพิ่มการทดสอบ visual regression

2. **ประสบการณ์การพัฒนา**:
   - เพิ่ม Storybook สำหรับเอกสารคอมโพเนนต์
   - ใช้กฎ ESLint เพื่อความสอดคล้อง
   - เพิ่ม pre-commit hooks ด้วย Husky

3. **การ Deploy**:
   - เพิ่ม CI/CD pipeline
   - ใช้การทดสอบอัตโนมัติ
   - เพิ่มการตรวจสอบประสิทธิภาพ
   - ตั้งค่าการติดตามข้อผิดพลาด (Sentry)

### ความปลอดภัย:
1. **Content Security Policy** (CSP)
2. **การฆ่าเชื้อ Input** สำหรับฟอร์มติดต่อ
3. **การจำกัดอัตรา** สำหรับการส่งฟอร์ม
4. **การบังคับใช้ HTTPS**

- **แอนิเมชันขั้นสูง** และองค์ประกอบแบบโต้ตอบ
- **ดีไซน์ตอบสนอง** ด้วยแนวทาง mobile-first
- **การเพิ่มประสิทธิภาพ** ผ่านแนวปฏิบัติที่ดีที่สุดของ React
